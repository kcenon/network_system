Detailed Description

Adapter that wraps messaging_udp_client to implement i_connection.

This adapter bridges the existing UDP client implementation with the new unified interface. Note that UDP is connectionless - "connect" simply sets the target endpoint, and the connection is considered "connected" when the client is running.

Thread Safety

Thread-safe. All methods can be called from any thread.

Ownership

The adapter owns the underlying client via shared_ptr for proper RAII.

UDP Semantics

  • connect() sets target endpoint and starts the client
  • is_connected() returns true when client is running
  • send() sends datagrams to the configured target endpoint
  • Data arrives via on_data callback
