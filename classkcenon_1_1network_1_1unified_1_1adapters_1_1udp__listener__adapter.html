Detailed Description

Adapter that wraps messaging_udp_server to implement i_listener.

This adapter bridges the existing UDP server implementation with the new unified interface. Note that UDP is connectionless - there are no real "connections" to accept. Instead, we track unique remote endpoints that send data to us.

Thread Safety

Thread-safe. All methods can be called from any thread.

Ownership

The adapter owns the underlying server via shared_ptr for proper RAII.

UDP Semantics

  • start() binds to a local port and begins receiving datagrams
  • Each unique sender endpoint is tracked as a virtual "connection"
  • on_accept callback is triggered when a new endpoint sends data
  • send_to() and broadcast() send datagrams to specific/all endpoints
