Detailed Description

Anti-replay protection for 0-RTT data.

Implements a time-based replay filter to prevent replay attacks on 0-RTT early data. Uses a sliding window approach based on client hello random values.

RFC 8446 Section 8

Servers that accept 0-RTT must implement anti-replay protection. This implementation uses a combination of:

  • Time-based window (reject old tickets)
  • Nonce tracking (reject duplicate tickets within window)

Thread Safety

All public methods are thread-safe.
